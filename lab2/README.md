# Лабораторна робота №2

## Розробка структури застосунку

### Вихідні дані

**Предмет:** Розробка програмного забезпечення на платформі Node.js  
**Назва проєкту:** Student Notes System  
**Тип застосунку:** CLI (консольний застосунок)

## 1. Загальний опис застосунку

Student Notes System — це застосунок на платформі Node.js, призначений для
збереження, перегляду та керування навчальними нотатками студентів.
Застосунок дозволяє організовувати нотатки за навчальними дисциплінами
та користувачами.

## 2. Основні компоненти (модулі)

### 2.1 CLI Module

Модуль взаємодії з користувачем через командний рядок.
Відповідає за прийом команд, валідацію введених даних та
передачу запитів у бізнес-логіку застосунку.

### 2.2 Notes Module

Модуль керування нотатками.
Відповідає за створення, редагування, видалення та перегляд нотаток,
а також за фільтрацію нотаток за дисциплінами та користувачами.

### 2.3 Subjects Module

Модуль керування навчальними дисциплінами.
Відповідає за створення та зберігання дисциплін, а також за
перевірку коректності дисципліни при створенні нотаток.

### 2.4 Users Module

Модуль керування користувачами.
Відповідає за зберігання інформації про користувачів
та зв’язок користувачів із нотатками.

### 2.5 Storage Module

Модуль збереження даних.
Відповідає за читання, запис, оновлення та видалення даних.
Ізолює спосіб збереження даних від бізнес-логіки застосунку.

## 3. Взаємодія компонентів

```text
Користувач
↓
CLI Module
↓
Notes Module ↔ Subjects Module
↓
Users Module
↓
Storage Module

```

CLI Module отримує команди користувача та передає їх відповідним модулям.
Усі операції з даними виконуються через Storage Module.

## 4. Дані та їх зв’язки (ER-діаграма)

### Сутності

**User**

- id
- name
- email

**Subject**

- id
- title

**Note**

- id
- title
- content
- createdAt
- userId
- subjectId

### Зв’язки

- один користувач (User) може мати багато нотаток (Note)
- одна дисципліна (Subject) може мати багато нотаток
- кожна нотатка належить одному користувачу та одній дисципліні

User (1) ─── (N) Note (N) ─── (1) Subject

## 5. Основні сценарії роботи застосунку

### Створення нотатки

Користувач вводить команду створення нотатки через CLI.
Notes Module створює нову нотатку та передає її для збереження у Storage Module.

### Перегляд нотаток

Користувач запитує список нотаток.
Застосунок отримує дані зі сховища та відображає їх користувачу.

### Редагування нотатки

Користувач обирає нотатку та змінює її вміст.
Оновлені дані зберігаються у сховищі.

### Видалення нотатки

Користувач видаляє нотатку.
Нотатка видаляється зі сховища даних.

### Агрегація даних

Застосунок може підраховувати кількість нотаток
за дисциплінами або за користувачами.

## 6. Висновок

У лабораторній роботі №2 було розроблено структуру застосунку
Student Notes System, описано основні модулі, їх взаємодію,
модель даних та ключові сценарії роботи системи.
